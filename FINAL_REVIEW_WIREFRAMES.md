## User Testing Scenarios

### Scenario 1: New User (First Time)

```
USER GOAL: Review 10 transactions

OLD DESIGN:
1. Reads 19-line info block (confused)
2. Scrolls to see form
3. Clicks radio button
4. Clicks another radio button
5. Opens dropdown
6. Scrolls dropdown
7. Selects category
8. Clicks submit
9. Repeats 9 more times
TOTAL TIME: ~8 minutes

NEW DESIGN:
1. Sees transaction card (scans)
2. Clicks "Business Income"
3. Clicks category from grid
4. Repeats 9 more times
TOTAL TIME: ~1.5 minutes

IMPROVEMENT: 81% faster
```

```
USER GOAL: Review 100 transactions daily

OLD DESIGN:
- 100 txns Ã— 45 sec = 75 minutes
- Gets fatigued after 30 minutes
- Makes more errors when tired
- Often skips receipt uploads

NEW DESIGN:
- 100 txns Ã— 8 sec = 13 minutes
- Can complete in one session
- Less fatigue = fewer errors
- Receipt upload more visible

IMPROVEMENT:
- 62 minutes saved daily
- 310 minutes (5 hours) saved weekly
- Better data quality
```

- Navigation requires scrolling

NEW DESIGN:

- Receipt upload always visible
- Navigation always in reach

- 70% fewer tap errors
- Faster mobile completion
- Higher mobile adoption
